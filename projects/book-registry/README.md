# book-registry

Личный реестр прочитанных книг.

Проект представляет собой лёгкое веб-приложение, в котором можно хранить список уже прочитанных книг, выставлять им оценки, фиксировать дату прочтения и оставлять комментарии. Приложение сделано как один компактный Node.js-сервис с фронтом и бэком в одном контуре.

## Что уже реализовано

На текущий момент приложение умеет:

- входить по паролю;
- хранить данные в `data/books.json`;
- показывать список книг;
- добавлять новые записи;
- редактировать существующие записи;
- удалять записи;
- сортировать список по дате прочтения по убыванию;
- фильтровать книги по названию и автору;
- показывать пагинацию;
- менять размер страницы (`10`, `25`, `50`, `100`);
- показывать базовую статистику;
- показывать график книг по годам;
- сворачивать и разворачивать блок статистики;
- показывать более компактный интерфейс с минималистичными карточками и иконками действий;
- работать в одном Docker-контейнере.

## Модель данных

Каждая запись о книге содержит:

- `id` — уникальный идентификатор;
- `title` — название книги;
- `author` — автор;
- `rating` — личная оценка от `1` до `10`;
- `finishedAt` — дата прочтения в формате `YYYY-MM-DD`;
- `comment` — комментарий.

Пример:

```json
{
  "id": "0d9e4b3e-6d7b-4c8b-9a6a-123456789abc",
  "title": "Dune",
  "author": "Frank Herbert",
  "rating": 9,
  "finishedAt": "2026-03-18",
  "comment": "Сильная атмосфера и масштаб мира."
}
```

## Хранение данных

Данные приложения хранятся в JSON-файле:

- `data/books.json`

Базовая структура файла:

```json
{
  "books": []
}
```

Для этого проекта `data/books.json` считается рабочим файлом данных и **отслеживается в git**, так как записи о книгах не являются секретом и должны легко переноситься на другую машину.

## Архитектура

Проект собран как один лёгкий сервис:

- **frontend** — HTML/CSS/JavaScript без тяжёлого фреймворка;
- **backend** — Node.js + Express;
- **storage** — `data/books.json`.

Приложение работает как единый контур:

- один процесс;
- один порт;
- фронт раздаётся тем же сервером;
- API обслуживается тем же приложением.

## Аутентификация

Используется минимальная парольная защита.

Принцип работы:

1. пользователь открывает страницу;
2. вводит пароль;
3. пароль отправляется на backend;
4. backend проверяет его;
5. при успехе backend устанавливает cookie-сессию;
6. дальнейшие запросы к API разрешаются только при валидной cookie.

### Правила безопасности

- пароль хранится только на backend через `.env`;
- frontend не содержит правильного пароля;
- cookie — `HttpOnly`;
- рекомендуется `SameSite=Strict`;
- флаг `Secure` управляется отдельно через `COOKIE_SECURE=true|false` и должен включаться только при реальном HTTPS.

## Реализованный API

### Аутентификация

- `POST /api/login` — вход и установка cookie-сессии;
- `POST /api/logout` — завершение сессии.

### Книги

- `GET /api/books` — получить список книг с пагинацией и фильтрами;
- `POST /api/books` — создать запись;
- `PUT /api/books/:id` — изменить запись;
- `DELETE /api/books/:id` — удалить запись.

### Статистика

- `GET /api/books/stats` — получить базовую статистику по реестру, включая разрез по годам.

## Список, фильтрация, сортировка и пагинация

По умолчанию список сортируется:

- по `finishedAt`
- по убыванию
- то есть сначала показываются самые недавно прочитанные книги

### Фильтрация

`GET /api/books` поддерживает параметры:

- `title`
- `author`

Фильтрация:

- работает по подстроке;
- регистр игнорируется;
- применяется до пагинации.

### Пагинация

`GET /api/books` поддерживает параметры:

- `page`
- `pageSize`

Допустимые размеры страницы:

- `10`
- `25`
- `50`
- `100`

Размер страницы по умолчанию:

- `10`

Пример ответа:

```json
{
  "books": [],
  "pagination": {
    "page": 1,
    "pageSize": 10,
    "totalItems": 0,
    "totalPages": 1
  },
  "filters": {
    "title": "",
    "author": ""
  }
}
```

## Реализованная статистика

Сейчас приложение показывает 4 базовых показателя:

- **Всего книг**
- **Средняя оценка**
- **Последняя прочитанная**
- **Книг в текущем году**

Дополнительно строится график:

- **книги по годам**

Блок статистики по умолчанию скрыт и раскрывается по кнопке.

## Интерфейс

Текущий UI устроен так:

- основной экран показывает список книг;
- форма добавления по умолчанию скрыта;
- кнопка **«Добавить книгу»** раскрывает форму;
- при редактировании книги форма открывается автоматически;
- статистика вынесена в отдельный раскрываемый блок;
- в списке есть локальные фильтры по названию и автору;
- действия у карточки книги оформлены иконками;
- оценка отображается более визуально, при этом сохраняет шкалу `1–10`;
- интерфейс адаптирован для мобильных экранов.

## Переменные окружения

Пример конфигурации лежит в `.env.example`.

Основные переменные:

- `PORT` — порт приложения;
- `APP_PASSWORD` — пароль входа;
- `SESSION_COOKIE_NAME` — имя cookie;
- `SESSION_TOKEN` — токен серверной сессии;
- `NODE_ENV` — режим окружения;
- `COOKIE_SECURE` — использовать ли `Secure` для cookie.

## Локальный запуск

```bash
cp .env.example .env
npm install
npm start
```

После запуска приложение обычно доступно на:

- `http://localhost:3000`

## Запуск в Docker

### Вариант через compose-файл

```bash
cp .env.example .env
cp data/books.example.json data/books.json
docker compose up --build -d
```

### Вариант одним контейнером

Проект также может быть поднят как один контейнер с внешним подключением файла данных `data/books.json`.

## Почему выбран JSON

JSON выбран как стартовый формат, потому что он:

- прост в чтении;
- легко редактируется вручную;
- хорошо живёт в git;
- не требует отдельной базы данных;
- позволяет позже без большой боли перейти на SQLite или другое хранилище.
